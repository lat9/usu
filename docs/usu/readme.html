Current Zen Cart Download is here.

See the plugin's GitHub repository for upcoming changes and bugfixes.

Ultimate URLs (USU) allows store owners to use alternate URLs. The URLs generated will include "human friendly" names in the current browser's requested language and contain shortened versions of certain parameters.

Note: If you are updating USU from a version prior to v3.0.0, additional care must be taken to ensure that the previous version's changes have been fully removed from your store's file-system … before you copy the updated USU files to your store's file-system. Refer to the Installation tab for details.

Features

  • Both the stock Zen Cart and alternative URLs can be used to access the pages.
  • One can enable automatic 301 redirects from older URLs (both stock and alternative) to the new URLs (both updated stock and alternative).
  • Support is included for generating URLs in the native language requested by the browser / customer.
  • For category, product, and review pages: one can choose to show just the product / category name or optionally prepend the name with the parent category's name (original or parent).
  • For category pages: one can choose to display the categories as directories (off, short, or full).
  • For EZ-Pages: the title of the EZ-Page is be used.
  • For all other pages: the page type will be used as the name (replaces "main_page=xyz" with "xyz").
  • One can optionally chose to have all alternate URLS end with a specific extension (.html, .htm).
  • One can configure sitewide string replacements using PCRE rules (regular expressions / preg_replace).
  • One can choose to strip all non-alphanumerical characters or just "special" characters (dash is excluded).
  • One can choose to "filter" out short words.
  • One can select which page types (main_page=xyz) will have rewritten URLs.
  • One can choose to have generated URLs cached. Caching is supported at multiple levels including the use of memory and database storage.
  • If one chooses to remove Ultimate URLs and switch back to the stock Zen Cart URLs in the future, leaving the .htaccess rules in place will allow both stock and alternative URLs to continue functioning.

Requirements

  • Working installation of Zen Cart 1.5.7a or later. Note: Zen Cart 1.5.7 (unsuffixed) has issues (if unpatched) with multiple observers watching the same event.
  • Supports PHP versions 5.6 through 8.2.
  • Secure FTP access to the Zen Cart installation (while you can use plain FTP it is not recommended)
  • An account to access Zen Cart's administrative interface
  • Knowledge of regular expressions if filtering names in URLs
  • Read and agree to the License

Plugin File Structure

  • .htaccess (Apache configuration directives)
  • /includes (storefront processing)
  • /YOUR_ADMIN (admin-level processing)
  • /docs readme (documentation)

Important Notes

Starting with v3.0.0, Ultimate URLs is, from a PHP-standpoint, a drop-in plugin! You do need to merge USU's .htaccess contents with your store's current file-contents.

Simply follow the instructions below to install or update USU for your store. This version of the plugin provides an additional tool to allow you to easily remove USU's files and database updates.

Note: If you are upgrading from a version of USU prior to v3.0.0, follow the steps outlined here to remove previous versions' changes to Zen Cart core-files, then proceed to Installation and Upgrades.

This section identifies the processes you'll use to initially install (or upgrade) USU or uninstall USU entirely.

Note: Be sure to review the pages identified in the Configuration :: Ultimate URLs :: Limit alternate URLS to the following pages setting! The default installation does not include entries for the ask_a_question, product_reviews, product_reviews_info or product_reviews_write pages. Without those entries, the USU version of those URLs will not be supplied!

  • Installation and Upgrades
  • Uninstalling

There are neither core-file overwrites nor template-overrides used by this plugin; you should always backup your cart's database and files prior to making any changes. Unzip the plugin's package file and then:

  1. Rename the YOUR_ADMIN directory to match your custom admin directory's name.
  2. Sign into your Zen Cart's admin panel.
  3. Copy the plugin's .php files to your store's root directory structure:
    1. /includes/auto_loaders/config.usu.php
    2. /includes/classes/usu.php
    3. /includes/classes/observers/UsuObserver.php
    4. /includes/extra_datafiles/usu.php
    5. /includes/init_includes/init_usu.php
    6. /YOUR_ADMIN/usu_uninstall.php
    7. /YOUR_ADMIN/includes/auto_loaders/config.usu.php
    8. /YOUR_ADMIN/includes/classes/observers/UsuAdminObserver.php
    9. /YOUR_ADMIN/includes/extra_datafiles/usu.php
    10. /YOUR_ADMIN/includes/functions/extra_functions/usu.php
    11. /YOUR_ADMIN/includes/init_includes/init_usu_admin.php
    12. /YOUR_ADMIN/includes/init_includes/init_usu_admin_install.php
    13. /YOUR_ADMIN/includes/init_includes/init_usu_admin_update.php
    14. /YOUR_ADMIN/includes/init_includes/init_usu_admin_update_from_ultimate_seo.php
    15. /YOUR_ADMIN/includes/languages/english/usu_configuration.php
    16. /YOUR_ADMIN/includes/languages/english/usu_uninstall.php
    17. /YOUR_ADMIN/includes/languages/english/extra_definitions/usu.php
    Note: The list above includes only the required files in USU's distribution. See the Supplied Files tab for additional information.
  4. If you are upgrading the plugin from a previous version, you've already incorporated the plugin's required changes to your site's .htaccess file; continue to the next step. Otherwise:
    • The .htaccess file in the root of USU's distribution contains directives for Apache web servers on how to handle incoming requests for URLs including those generated by Ultimate URLs. If your webhost uses an Apple OSX-, UNIX-, or BSD-based operating system you may need to change some settings to allow you to see the files in this folder. Look for an option to "show hidden files". By default, these operating systems can be configured to hide files that start with a dot.
    • Edit the supplied sample .htaccess file with a plaintext editor (such as notepad). Adjust the Apache directives (especially RewriteBase) as needed for the host where your Zen Cart is installed — pay particular attention to the notes and comments inside the sample .htaccess file.
    • If an existing .htaccess file exists at the root of your Zen Cart installation, you should merge the directives from that pre-existing file with those required for USU.
    • Make a copy of any pre-existing .htaccess for your records.
    • Upload the edited .htaccess file to the root of your Zen Cart installation.
  5. Click the "Admin Home" link in your Zen Cart's top admin menu. This action will cause the plugin's installation script to run. You'll see an addition to your Zen Cart's Configuration menu entitled Ultimate URLs. The plugin will not become active until you set the plugin's Enable alternate URLs? setting to true.

Navigate to your Zen Cart admin's Extras :: Uninstall Ultimate URLs tool and follow the instructions. That tool will delete all the plugin's added files and database changes … leaving the site's .htaccess intact, since you might have other customizations that would be lost upon delete.

You can view the details of these changes on the plugin's GitHub repository.
